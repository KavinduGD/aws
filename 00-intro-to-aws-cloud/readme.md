# Cloud

## Cloud models

1. Cloud (public cloud) - A cloud computing environment that is owned and operated by a third-party cloud service provider and delivers computing resources over the internet to multiple customers. Examples include Amazon Web Services (AWS), Microsoft Azure, and Google Cloud Platform (GCP).
2. On-premises (private cloud) - A cloud computing environment that is owned and operated by an organization for its own use. It is typically deployed within the organization's data center and provides more control and security than a public cloud.
3. Hybrid cloud - A cloud computing environment that combines a public cloud and a private cloud, allowing data and applications to be shared between them. This model provides flexibility and optimization of existing infrastructure, business, and compliance requirements.

## Benefits of cloud

1. Trade upfront capital expenses(Capital Expenditures) for variable operating expenses(Operating Expenses) - Cloud computing allows organizations to avoid the upfront costs of purchasing and maintaining hardware and software, and instead pay for resources on a pay-as-you-go basis.

2. Stop focusing on the data center

3. Stop guessing capacity - Cloud computing allows organizations to scale their resources up or down as needed, eliminating the need to guess how much capacity they will need in the future.

4. Benefits from massive economies of scale - Cloud service providers can leverage their large scale to offer lower prices and better performance than organizations could achieve on their own. (produce a high volume of goods or services, causing the average cost per unit to drop significantly.). Also aws provide lower prices when we use more resources, which is called volume discounts. For example, if we use higher amount of resources in s3, we can get lower prices per GB stored.

5. Increase provisioning speed and agility - Cloud computing allows organizations to quickly provision and deploy resources, enabling them to respond faster to changing business needs and market conditions.

6. Go global in minutes - Cloud computing allows organizations to easily deploy their applications and services in multiple regions around the world, enabling them to reach a global audience and improve performance for users in different locations.

## Cloud design principles

1. Design for failure - Cloud applications should be designed to handle failures gracefully, such as by using redundant resources and implementing failover mechanisms.
2. Implement elasticity - Cloud applications should be designed to scale up or down as needed, allowing them to handle changes in demand and optimize resource usage.
3. Decouple components - Cloud applications should be designed to decouple components, allowing them to be developed, deployed, and scaled independently. If one component fails, it should not affect the entire application.
4. Think parallel - Cloud applications should be designed to take advantage of parallel processing, allowing them to perform tasks more efficiently and improve performance.

## Economy model of aws

1. Free tier - AWS offers a free tier that provides limited access to certain services for new customers for 12 months. Some services are always free, such as AWS Lambda, Amazon S3, and Amazon DynamoDB. This allows customers to try out AWS services without incurring any costs.
2. On-demand pricing - AWS offers on-demand pricing for most of its services, which allows customers to pay for resources on a pay-as-you-go basis. This means that customers only pay for the resources they use, and there are no upfront costs or long-term commitments.
3. Reservations - AWS offers reserved instances for certain services, which allow customers to reserve capacity for a specific period of time (e.g., 1 or 3 years) in exchange for a discounted hourly rate. This can provide cost savings for customers who have predictable workloads and can commit to using resources for a longer period of time.
4. Volume discounts - AWS offers volume discounts for certain services, which provide lower prices as customers use more resources. For example, if a customer uses a higher amount of storage in Amazon S3, they can get lower prices per GB stored.
5. Price drop - AWS regularly reduces prices for its services, which can provide cost savings for customers over time. This is due to the economies of scale that AWS achieves as it grows its customer base and infrastructure.
