# Cloud

## Cloud models

1. Cloud (public cloud) - A cloud computing environment that is owned and operated by a third-party cloud service provider and delivers computing resources over the internet to multiple customers. Examples include Amazon Web Services (AWS), Microsoft Azure, and Google Cloud Platform (GCP).
2. On-premises (private cloud) - A cloud computing environment that is owned and operated by an organization for its own use. It is typically deployed within the organization's data center and provides more control and security than a public cloud.
3. Hybrid cloud - A cloud computing environment that combines a public cloud and a private cloud, allowing data and applications to be shared between them. This model provides flexibility and optimization of existing infrastructure, business, and compliance requirements.

## Benefits of cloud

1. Trade upfront capital expenses(Capital Expenditures) for variable operating expenses(Operating Expenses) - Cloud computing allows organizations to avoid the upfront costs of purchasing and maintaining hardware and software, and instead pay for resources on a pay-as-you-go basis.

2. Stop focusing on the data center

3. Stop guessing capacity - Cloud computing allows organizations to scale their resources up or down as needed, eliminating the need to guess how much capacity they will need in the future.

4. Benefits from massive economies of scale - Cloud service providers can leverage their large scale to offer lower prices and better performance than organizations could achieve on their own. (produce a high volume of goods or services, causing the average cost per unit to drop significantly.)

5. Increase provisioning speed and agility - Cloud computing allows organizations to quickly provision and deploy resources, enabling them to respond faster to changing business needs and market conditions.

6. Go global in minutes - Cloud computing allows organizations to easily deploy their applications and services in multiple regions around the world, enabling them to reach a global audience and improve performance for users in different locations.
